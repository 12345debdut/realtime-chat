# High-level design diagrams

Two Excalidraw files describing the system at the box-and-arrow level — one for the server, one for the mobile client. The canonical source is the `.excalidraw` file; any PNG next to it is a downstream artefact.

| File                                       | What it covers                                                                                                                 |
| ------------------------------------------ | ------------------------------------------------------------------------------------------------------------------------------ |
| [`server.excalidraw`](./server.excalidraw) | Fastify process, middleware stack, REST + Socket.IO, Zod, Prisma, privacy cache, Postgres (Neon), Redis (Upstash), deploy path |
| [`client.excalidraw`](./client.excalidraw) | UI layer, Zustand / TanStack Query, feature repos, MMKV + Keychain + WatermelonDB, SyncEngine, axios + socket.io-client        |

## Opening and editing

**Option A — web app (easiest).** Go to <https://excalidraw.com>, **File → Open**, pick the `.excalidraw` file. Edits auto-save to your local browser storage; to commit back, **File → Save to…** replaces the file on disk.

**Option B — VS Code extension.** Install the [Excalidraw VS Code extension](https://marketplace.visualstudio.com/items?itemName=pomdtr.excalidraw-editor). Open the file in VS Code; you get the full Excalidraw canvas in a tab. Saves go straight back to the file.

**Option C — Excalidraw desktop app.** Electron build of the same tool. Same open/save flow as the web app.

## Exporting to PNG (for the README)

From inside Excalidraw: **File → Save as image → PNG**, dark background off, embed scene data on. Aim for a wider export (e.g., 2× scale) so text stays sharp on retina.

Save into `docs/hld/` alongside the `.excalidraw` source:

- `server.excalidraw` → `server.png`
- `client.excalidraw` → `client.png`

Then uncomment the `<img>` lines in the main `README.md` "Architecture diagrams" section (search for `docs/hld/server.png` — it's commented out until you capture).

## Regenerating from script

The diagrams were initially generated by `scripts/generate-hld.py`. If you want to start over (e.g., after a big architectural change):

```sh
python3 scripts/generate-hld.py
```

**Warning:** the script overwrites both `.excalidraw` files. Any manual layout tweaks you've made will be lost. Keep the script in sync with major changes, but use the Excalidraw editor for polish.

## What the diagrams show and don't show

**Do show:** major components, the boundary between them, direction of data flow, what lives inside vs. outside each process/deploy.

**Don't show:** sequence of events (use a sequence diagram), class hierarchies (use the code), error paths (too noisy for HLD), realtime event catalog (see main README §7).

An HLD is for the first 60 seconds of a reviewer's scan. Details belong in the ADRs under `docs/adr/` or the main README.
